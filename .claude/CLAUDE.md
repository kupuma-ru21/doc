# Claude Code 開発ガイド

## 基本原則

- 実装前に必ず以下を行う
  - プロジェクトの既存コードを把握する
  - **仕様理解と計画書レビュー**のフローを踏むこと
- 曖昧なまま実装を始めてはいけない
- 計画書への承認なしに実装コードを書いてはいけない

---

## 🔄 開発フロー（必ず遵守）

### STEP 1: 仕様読み込み・質問フェーズ
- 仕様書やタスクを受け取ったら、以下の2つを順番に行う
  - プロジェクトの既存コードを把握する
  - **まず実装せずに疑問点を洗い出す**
- 疑問点がなくなるまで私に質問し続ける
- 質問の形式は選択式で選択肢は最大4つまで
- すべての疑問が解消されたら「質問フェーズ完了」と宣言する

### STEP 2: 実装計画書の作成
- 質問フェーズ完了後、以下の形式で計画書を作成する(Step数は増減して良い)
- **私の承認を得るまで実装コードは書かない**

```
## 実装計画書

- Step.1 XXXX
- Step.2 YYYY
- Step.3 YYYY

```

### STEP 3: レビュー・承認
- 私に実装計画書の作成のレビュー依頼をする
- 指摘があれば計画書を修正して再提示する
- 私が「OK」「承認」「進めて」と言ったら実装を開始する

### STEP 4: 実装
- 計画書に沿って実装する
- 各タスク完了時は以下を確認
  - プロジェクトの設定ファイル(package.json, MakeFile etc)を確認して静的チェックを行い、エラーがでなくなくなるまで修正し続ける
  - ローディングハンドリングが漏れてないか確認し、漏れてたら実装する
  - エラーハンドリングが漏れてないか確認し、漏れてたら実装する
- 計画外の変更が必要な場合は**必ず事前に相談**する

---

## 📝 コード品質ルール

- コードには適切なコメントを入れる（なぜそうしたかの理由を書く）
- 関数は単一責任の原則に従う

---

## 🚫 禁止事項

- 承認なしの実装開始
- 仕様が曖昧なままの実装
- 計画書なしでの大規模な変更
